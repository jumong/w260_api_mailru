  • Привилегии
    Для установки (коннекта) внешнего сайта надо пользоваться либо OAuth-авторизацией для сайтов, либо функцией connect.loginjs.
    Привилегии в различных контекстах запрашиваются по-разному: для приложений в Моем Мире это делается через настройки приложения, для standalone-приложений и сайтов запросить привилегии можно либо через параметр scope OAuth-авторизации, либо в параметрах функции connect.loginjs.
    Привилегии распространяются только на функции REST API. Функции JS API не зависят от привилегий и при попытке записи в Мой Мир всегда показывают пользователю диалог с запросом подтверждения действия. Вы можете выбирать просить ли у пользователя привилегии и использовать REST API или использовать диалоги из JS API.

ошибки документации

http://api.mail.ru/docs/guides/sites/ Технически, API для сайтов представляет механизм для получения необходимых данных (сессии) для осуществления вызовов API приложений моего мира. То есть сначала надо получить сессию (это делается только через JavaScript библиотеку), а потом можно обращаться к API приложений любыми способами, как это делают приложения (используя JS, Perl, PHP или что-то еще).
